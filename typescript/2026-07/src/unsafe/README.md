# unsafe/

TypeScript のコア言語機能だけでは表現しにくい型ユーティリティを定義する例外領域（アーキテクチャルール 3.7・6.1）。**厳しいレビューが要求される代わりに、言語機能の不足を自前で拡張する**位置付けであり、通常実装の型エラー回避に使う場所ではない。

## 依存方向（非対称）

- ここから他のディレクトリへの依存は一切禁止する（`unsafe -> 依存なし`）。
- 他の領域からここへの依存は、言語機能の延長として**どの領域からも許可される**（ルール 4）。

## 許可している例外

- 型アサーション（`as` / `<T>value`）と型レベルの `any` を、型ユーティリティの定義に必要な範囲でのみ使用できる（ルール 6.1）。
- 値の ambient 宣言（`declare const` によるユニークシンボルの宣言など）。通常領域では禁止（ルール 6.1）だが、ここでのみ型ユーティリティの定義に必要な範囲で使用できる。

## 許可していないもの

- `null`・`enum`・型述語・非nullアサーション・definite assignment assertion（ここでも禁止のまま）
- `declare module`・`declare global`・`namespace`（宣言マージの侵入口。ここはどの領域からも依存されうるため影響が全域に及ぶ。ここでも禁止のまま、ルール 6.8.3・6.1）
- 業務ルール、モジュール固有の型、I/O（DB・HTTP・外部API・環境変数）に関わるコード
- 実行時バリデーションの回避、`unknown` や外部レスポンスを検証なしで任意の型へ変換する関数

## 典型的なアンチパターン（禁止）

通常領域の `as` 禁止を迂回するための「便利な」キャスト関数を、ここに置いてはならない（ルール 3.7.1）。名前や意図にかかわらず、以下はすべて `as` を関数に包んで全域へ再輸出しているだけである。

```ts
export function usefulCast<T, U>(value: T): U;          // 任意→任意のキャスト
export function parseAs<T>(text: string): T;            // 検証なしで外部データを内部型に見せかける
export function definitely<T>(value: T | undefined): T; // `!` の言い換え
export function asInvoice(row: unknown): Invoice;       // 業務型の捏造（依存方向違反でもある）
```

判定基準: **入力に対して実行時に確かめていない、入力より強い型の主張を返す関数は禁止。** 戻り値にしか現れない型パラメータを呼び出し側が選べる関数、および Union の一部（`| undefined` など）を検証なしに削る関数が典型である。対比として `brand<T, B>` は幻影タグを付けるだけで、値の構造的な型は入力のまま保存される。ここの Lint は `as` を緩和しているため機械的には検出されない。レビューで必ずこの基準に照らすこと。

## 安全性の担保

- 公開APIごとに、なぜ通常の型表現で足りないのか・どのリスクを受け入れているのかをコメントまたはREADMEで記録する（ルール 3.7・16.3）。
- 実行時テストに加え、可能な限り型テストで検証する（ルール 9.5）。
